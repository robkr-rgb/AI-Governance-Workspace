Abundly Refactor - Build Scope

Generated from the attached dossier plus public Abundly docs. Built as a local governed-agent prototype, not as a production-certified platform.

Implemented Product Slice

Runtime

Agent turns with traceability

Each turn records attack detection, context assembly, planning, guardrail checks, credential injection, dispatch, audit and credits.

Governance

Policy below the model

External email and HTTP actions are approved or blocked by platform code, not by prompt wording.

Vault

Post-planning credential injection

Secrets are stored in a local AES-GCM vault and referenced after planning. Secret values are excluded from traces and audit messages.

Evals

Validation before confidence

Evals check required and forbidden tool dispatch, including the prompt-injection block path.
