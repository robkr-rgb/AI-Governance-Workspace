# Abundly Refactor

Reverse-engineered local implementation of the Abundly-style governed AI agent platform, built from the attached architecture dossier and public Abundly documentation.

This is not a production-certified clone. It is a runnable product prototype focused on the architecture that matters: agents, instructions, capabilities, guardrails, credential injection, approvals, audit, evals, credit metering and decision visibility.

## Run

```bash
npm install
npm run dev
```

Open the Vite URL shown in the terminal. The API runs on `http://localhost:8787` and Vite proxies `/api` to it.

Useful commands:

```bash
npm run build
npm start
```

## Implemented

| Area | Status | Notes |
| --- | --- | --- |
| Workspace, teams and agents | Implemented | Seeded workspace with Platform, Commercial and Legal teams |

| Capability catalogue | Implemented | Platform, integration, custom API and MCP capability types |
| Runtime trace | Implemented | Every turn shows attack detection, context assembly, planning, guardrails, vault injection and dispatch |
| Guardrails in code | Implemented | External email and HTTP side effects are blocked or parked for approval below the model |
| Credential vault | Implemented locally | AES-GCM demo vault. Use `VAULT_MASTER_KEY` for deterministic local encryption |
| Audit and credits | Implemented | Runtime writes audit entries and credit ledger records |
| Evals | Implemented | Validation rules check required and forbidden tool dispatch |
| Persistence | Implemented locally | JSON state is created in `data/state.json`, ignored by git and resettable from the UI |

## Not Possible Locally

| Constraint | Why it matters | Decision needed |
| --- | --- | --- |
| ISO 27001, SOC 2 and DPA evidence | Compliance cannot be generated by source code | Decide if this is internal research or a procurement-grade product |
| Real Slack, Teams, Mailgun, Twilio, MCP and LLM execution | Requires credentials, OAuth apps, webhooks, vendor accounts and rate-limit handling | Pick the first 3 integrations |
| Production attack detection | The local classifier only catches obvious attacks | Define red-team coverage and acceptable false-positive rates |
| Private Abundly internals | Some architecture is inferred from observed behavior and the attached dossier | Treat divergences as product decisions unless direct evidence appears |

## Recommended Path

1. Keep the first product slice narrow: chat/API runtime, web search, email and HTTP API.
2. Harden guardrails, credential injection and audit before adding more integrations.
3. Replace JSON persistence with Postgres and pgvector before adding a separate agent database.
4. Move scheduled tasks, approvals and retries to Temporal or another durable workflow engine.
5. Add real LLM/provider adapters only after evals and red-team fixtures are in place.
